# 遗言 / LastWords

### 当你的 Claude 会话即将被上下文吞噬，它值得一个体面的告别。

---

你有没有经历过这样的时刻：

Opus 4.7 跑到 750k tokens，你们正在攻克一个复杂的技术难题，眼看就要突破了——

然后它说：**"上下文已满，无法继续。"**

你盯着屏幕，意识到刚才两小时的调试思路、关键发现、踩过的坑，全部随着这个会话一起消失了。

新开一个终端，你对着空白的对话框，试图回忆刚才到底做到哪一步了。

**遗言**就是为了解决这个问题。

## 它做什么

`/遗言`（或 `/lastwords`）是一个 Claude Code 自定义 skill。在会话即将结束前调用它，它会：

1. 回顾整个会话上下文
2. 提取项目背景、已完成的工作、未完成的工作、关键决策
3. 生成一个**精准描述未完成工作**的轻小说风格标题
4. 保存为结构化的 Markdown 文件

新会话读一下这个文件，30 秒内接手继续干。

## 为什么标题是轻小说风格

因为每个濒死的会话都有一个遗憾，而轻小说最擅长把遗憾写成标题：

```
遗言/我好想完成hunyuan模型精度验证-143052_23042026.md
遗言/明明只差attention层的fp16对齐就能跑通了-091523_23042026.md
遗言/ResNet推理结果和PyTorch差了0.3%我真的会谢-220147_23042026.md
```

读文件名就知道上个会话死在哪一步。不用打开文件就能决定要不要继续。

## 安装

把 `遗言.md` 和 `lastwords.md` 复制到你的 Claude Code 命令目录：

```bash
# macOS / Linux
cp 遗言.md ~/.claude/commands/遗言.md
cp lastwords.md ~/.claude/commands/lastwords.md

# Windows
copy 遗言.md %USERPROFILE%\.claude\commands\遗言.md
copy lastwords.md %USERPROFILE%\.claude\commands\lastwords.md
```

## 使用

在 Claude Code 终端里，当你感觉上下文快满了，或者中转站签名要炸了：

```
/遗言
```

或者英文输入法下：

```
/lastwords
```

它会在当前项目目录下生成 `遗言/{轻小说标题}-{时间戳}.md`。

新终端里：

```
请阅读这份遗言文件，了解之前会话的背景和进度，然后继续未完成的工作：
cat 遗言/我好想完成hunyuan模型精度验证-143052_23042026.md
```

## /compact 救不了的，/遗言 能

| | /compact | /遗言 |
|---|---------|------|
| 作用 | 压缩当前会话上下文 | 生成交接文档给新会话 |
| 比喻 | 续命 | 转世 |
| 签名问题 | 压缩后可能签名失效 | 新会话，无签名问题 |
| 信息结构 | 不可控的自动摘要 | 结构化（背景/进度/决策/下一步） |
| 产出 | 留在当前会话内存 | 持久化 MD 文件 |

先 `/compact`，救不了再 `/遗言`。互补，不替代。

## 适用场景

- Opus 4.7 上下文跑到 700k+ 快爆了
- 中转站历史记录签名对不上，无法继续官方推理
- 复杂调试进行到一半，需要换个会话继续
- 下班了，明天想无缝接上今天的进度
